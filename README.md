# ارسال پیامک OTP با پایتون — آرتا پیامک

این مخزن شامل نمونه‌ای ساده و کاربردی برای ارسال [پیامک OTP](https://artapayamak.com/otp-sms/) با زبان پایتون است. در این مثال، درخواست ارسال پیامک از طریق API و با استفاده از کتابخانه `requests` به سرویس IPPanel Edge فرستاده می‌شود.

[آرتا پیامک](https://artapayamak.com/) ارائه‌دهنده پنل ارسال پیامک و راهکارهای پیامکی برای توسعه‌دهندگان و کسب‌وکارهای ایرانی است. با استفاده از API آرتا پیامک می‌توانید قابلیت‌هایی مانند ارسال رمز یک‌بارمصرف، پیامک‌های خدماتی و اطلاع‌رسانی‌های سیستمی را به نرم‌افزار خود اضافه کنید.

## کاربرد این نمونه

این نمونه برای توسعه‌دهندگانی مناسب است که می‌خواهند ارسال کد تأیید را در پروژه پایتون خود پیاده‌سازی کنند. از این قابلیت می‌توان در فرایندهای زیر استفاده کرد:

- ورود یا ثبت‌نام با شماره موبایل
- تأیید شماره تلفن همراه
- بازیابی رمز عبور
- تأیید تراکنش یا عملیات حساس
- احراز هویت دومرحله‌ای

برای آشنایی بیشتر با این سرویس، صفحه [API ارسال پیامک OTP](https://artapayamak.com/sms-otp-api/) را مطالعه کنید.

## پیش‌نیازها

## تنظیم توکن API

برای حفظ امنیت، توکن API نباید مستقیماً داخل سورس‌کد نوشته شود. توکن را در متغیر محیطی `IPPANEL_API_TOKEN` قرار دهید.

### Linux و macOS

```bash
export IPPANEL_API_TOKEN="your_api_token_here"
```

### Windows PowerShell

```powershell
$env:IPPANEL_API_TOKEN="your_api_token_here"
```

> نکته: تابع `os.environ.get()` متغیرهای محیطی سیستم را می‌خواند و فایل `.env` را به‌صورت خودکار بارگذاری نمی‌کند. برای استفاده مستقیم از فایل `.env` باید کتابخانه‌ای مانند `python-dotenv` را نصب و پیکربندی کنید.

## نمونه کد ارسال OTP

فایل زیر را با نام `send_otp.py` ایجاد کنید:

```python
import os

import requests


url = "https://edge.ippanel.com/v1/api/send"

api_token = os.environ.get("IPPANEL_API_TOKEN")

if not api_token:
    raise RuntimeError(
        "The IPPANEL_API_TOKEN environment variable is not set."
    )

payload = {
    "sending_type": "pattern",
    "from_number": "+983000505",
    "code": "xxxxxxxxxxxxxxx",
    "recipients": ["+989120000000"],
    "params": {
        "code": "458921"
    }
}

headers = {
    "Authorization": api_token,
    "Content-Type": "application/json"
}

try:
    response = requests.post(
        url,
        json=payload,
        headers=headers,
        timeout=15
    )
    response.raise_for_status()

    try:
        print(response.json())
    except requests.exceptions.JSONDecodeError:
        print(response.text)

except requests.exceptions.RequestException as error:
    print(f"OTP request failed: {error}")
```

## توضیح خط‌به‌خط کد

برای جلوگیری از به‌هم‌ریختگی نمایش متن فارسی و عبارت‌های فنی، هر قسمت از کد به‌صورت جداگانه توضیح داده شده است.

### ۱. وارد کردن کتابخانه‌ها

```python
import os
import requests
```

ماژول `os` برای خواندن توکن API از متغیرهای محیطی استفاده می‌شود.

کتابخانه `requests` وظیفه ارسال درخواست HTTP به API را بر عهده دارد.

### ۲. مشخص کردن آدرس API

```python
url = "https://edge.ippanel.com/v1/api/send"
```

متغیر `url` آدرس endpoint مربوط به ارسال پیامک را نگه می‌دارد. درخواست `POST` به این آدرس ارسال خواهد شد.

### ۳. خواندن توکن API

```python
api_token = os.environ.get("IPPANEL_API_TOKEN")
```

توکن API از متغیر محیطی `IPPANEL_API_TOKEN` خوانده می‌شود. با این روش، اطلاعات حساس داخل سورس‌کد قرار نمی‌گیرند.

### ۴. بررسی وجود توکن

```python
if not api_token:
    raise RuntimeError(
        "The IPPANEL_API_TOKEN environment variable is not set."
    )
```

این قسمت بررسی می‌کند که توکن API تنظیم شده باشد. اگر توکن وجود نداشته باشد، برنامه پیش از ارسال درخواست متوقف می‌شود و پیام خطای مشخصی نمایش می‌دهد.

### ۵. ساخت اطلاعات درخواست

```python
payload = {
    "sending_type": "pattern",
    "from_number": "+983000505",
    "code": "xxxxxxxxxxxxxxx",
    "recipients": ["+989120000000"],
    "params": {
        "code": "458921"
    }
}
```

متغیر `payload` اطلاعات موردنیاز برای ارسال OTP را در قالب یک دیکشنری پایتون نگه می‌دارد.

### ۶. تعیین نوع ارسال

```python
"sending_type": "pattern"
```

مقدار `pattern` مشخص می‌کند که پیامک با استفاده از الگوی ازپیش‌تعریف‌شده ارسال شود. ارسال الگویی برای پیامک‌های OTP و پیام‌های خدماتی کاربرد دارد.

### ۷. تعیین شماره فرستنده

```python
"from_number": "+983000505"
```

مقدار `from_number` شماره‌ای است که پیامک از طریق آن ارسال می‌شود. این مقدار را با شماره فرستنده فعال حساب خود جایگزین کنید.

### ۸. تعیین شناسه الگو

```python
"code": "xxxxxxxxxxxxxxx"
```

این `code` شناسه الگوی پیامکی ثبت‌شده در پنل است. مقدار نمونه را با شناسه واقعی الگوی تأییدشده خود جایگزین کنید.

این فیلد با مقدار OTP داخل `params` تفاوت دارد؛ فیلد بالایی شناسه الگو است، اما مقدار موجود در `params` رمز یک‌بارمصرفی است که برای کاربر ارسال می‌شود.

### ۹. تعیین شماره گیرنده

```python
"recipients": ["+989120000000"]
```

فیلد `recipients` فهرست شماره‌های دریافت‌کننده پیامک را مشخص می‌کند. شماره موبایل باید با فرمت مورد قبول API وارد شود.

در این نمونه، شماره با پیش‌شماره بین‌المللی ایران یعنی `+98` نوشته شده است.

### ۱۰. مقداردهی متغیرهای الگو

```python
"params": {
    "code": "458921"
}
```

فیلد `params` مقادیر متغیرهای داخل الگوی پیامک را تعیین می‌کند.

در این نمونه، متغیر `code` داخل الگو با مقدار `458921` جایگزین می‌شود. در یک پروژه واقعی، این مقدار باید به‌صورت امن و پویا تولید شود.

نام کلیدهای `params` باید دقیقاً با نام متغیرهای تعریف‌شده در الگوی پیامک مطابقت داشته باشد.

### ۱۱. ساخت هدرهای درخواست

```python
headers = {
    "Authorization": api_token,
    "Content-Type": "application/json"
}
```

هدر `Authorization` توکن API را برای احراز هویت درخواست ارسال می‌کند.

هدر `Content-Type` نیز مشخص می‌کند که محتوای درخواست در قالب JSON فرستاده می‌شود.

### ۱۲. ارسال درخواست

```python
response = requests.post(
    url,
    json=payload,
    headers=headers,
    timeout=15
)
```

متد `requests.post()` درخواست را به API ارسال می‌کند.

پارامتر `json` دیکشنری پایتون را به JSON تبدیل می‌کند.

پارامتر `headers` اطلاعات احراز هویت و نوع محتوای درخواست را ارسال می‌کند.

پارامتر `timeout` حداکثر زمان انتظار برای دریافت پاسخ را روی ۱۵ ثانیه قرار می‌دهد.

### ۱۳. بررسی وضعیت پاسخ

```python
response.raise_for_status()
```

این دستور وضعیت پاسخ HTTP را بررسی می‌کند. اگر سرور یک خطای HTTP مانند `400`، `401` یا `500` برگرداند، یک exception ایجاد می‌شود.

### ۱۴. نمایش پاسخ API

```python
try:
    print(response.json())
except requests.exceptions.JSONDecodeError:
    print(response.text)
```

اگر پاسخ API در قالب JSON باشد، با `response.json()` خوانده و چاپ می‌شود.

اگر پاسخ JSON معتبر نباشد، محتوای خام پاسخ با `response.text` نمایش داده می‌شود.

### ۱۵. مدیریت خطاهای درخواست

```python
except requests.exceptions.RequestException as error:
    print(f"OTP request failed: {error}")
```

این بخش خطاهای مرتبط با شبکه، پایان زمان انتظار و پاسخ‌های ناموفق HTTP را مدیریت می‌کند تا برنامه بدون نمایش خطای کنترل‌نشده متوقف نشود.

## شخصی‌سازی نمونه

پیش از اجرای کد، مقادیر نمونه زیر را با اطلاعات واقعی حساب خود جایگزین کنید:

| مقدار نمونه | کاربرد |
|---|---|
| `+983000505` | شماره فرستنده فعال |
| `xxxxxxxxxxxxxxx` | شناسه الگوی تأییدشده |
| `+989120000000` | شماره موبایل گیرنده |
| `458921` | کد یک‌بارمصرف تولیدشده توسط برنامه |

توکن API نباید در این جدول یا داخل سورس‌کد قرار بگیرد و باید از متغیر محیطی خوانده شود.

- توکن API را مستقیماً داخل سورس‌کد ننویسید.

- توکن API را در لاگ‌ها و پیام‌های خطا نمایش ندهید.

## خطاهای رایج

- **توکن نامعتبر:** مقدار `IPPANEL_API_TOKEN` را بررسی کنید.
- **شناسه الگوی اشتباه:** مقدار فیلد بالایی `code` باید شناسه الگوی تأییدشده باشد.
- **نام متغیر اشتباه:** کلیدهای `params` باید با متغیرهای الگو یکسان باشند.
- **شماره فرستنده نامعتبر:** مقدار `from_number` باید در حساب شما فعال باشد.
- **فرمت اشتباه شماره:** شماره گیرنده را با فرمت مورد قبول API ارسال کنید.
- **پاسخ `401`:** اطلاعات احراز هویت یا توکن API معتبر نیست.
- **پاسخ `422`:** یکی از مقادیر درخواست نامعتبر یا ناقص است.
